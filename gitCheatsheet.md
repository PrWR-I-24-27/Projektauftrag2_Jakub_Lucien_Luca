# Git Cheatsheet

## Start
- `Scripts\activate` → virtuelle Umgebung starten  
- 
- `git branch` → zeigt, auf welchem Branch du bist  
- `git checkout <branchname>` → zu einem anderen Branch wechseln  
- `git pull` → die neuesten Änderungen runterladen  

---

## Während der Arbeit
- `git status` → zeigt, was sich geändert hat  
- `git add .` → alle Änderungen merken  
- `git commit -m "Nachricht"` → speichern mit kurzer klarer Nachricht was gemacht worden ist

---

## Am Ende
- Prüfen, dass alles gespeichert ist  
- `git push` → Änderungen hochladen  

---

## Git Best Practices
- Oft Commits machen
- Klare Commitsmessages
    - Im Imperativ
- Branches benutzen 
    - Z.B. `bugfix/Entwicklungsplot`
- Pull vor dem Push
